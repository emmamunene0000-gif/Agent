## DAY 1 — MONDAY | CONCEPT: AUGMENTED INTELLIGENCE — THE OPERATOR'S ROLE

### What the Agent Is and Is Not

The Absolute Dollar Agent is an augmented intelligence system — a glass box that externalizes a complete multi-timeframe trading framework and fires structured signal briefs. It is not:
- An automated trading bot (it does not execute trades)
- An infallible signal service (no system is)
- A substitute for operator judgment

The agent is:
- A HUD (Heads-Up Display) communicating market state in real time
- A structured decision support system across 17 subsystems
- A glass box — the operator can see exactly why every output was produced

### When the Operator Should Override the Agent

The confluence gate (Week 4) already includes operator judgment at Check 5 (M15 structure) and Check 6 (ATR + timing). But there are additional scenarios where an operator should override or delay a signal:

**Override SKIP scenarios:**
1. Major economic event in the next 15 minutes (news blackout)
2. The BSL above a buy entry is clearly unswept and within easy reach
3. M15 has just printed a bearish CHoCH conflicting with a buy signal
4. Price is at a known institutional level (round number, weekly pivot) with no confirmation
5. Session context is wrong — Asia signal, but London open is 20 minutes away (wait)

**Override EXECUTE scenarios (rare, expert-level):**
1. Score is 3.5 (just below the +4 threshold) but manual chart read shows extremely clean structure
2. Signal fired during Asia but operator can see London killzone entry forming — wait for killzone and re-evaluate
3. Note: these overrides are only valid after Week 4 mastery. Beginners do not override.

### The Agent as Partner

The agent fires signals based on quantitative rules. The operator brings contextual judgment:
- Session awareness (is this the right time?)
- News calendar awareness (is there risk nearby?)
- HTF context the agent scores but the operator visualizes
- Personal capital management (is this the right trade size given account state?)

Together: quantitative precision (agent) + contextual judgment (operator) = a trading operation that is greater than either alone.

## DAY 2 — TUESDAY | CHART: THE COMPLETE SESSION WORKFLOW

### The Operator's Daily Routine — Full Protocol

```

[A] H4 READ
  - What session closed last? Note the H4 candle direction.
  - Asia H4 closed: BULL / BEAR / DOJI?
  - Weekly bias (from Monday H4 open direction): BULL / BEAR?
  - H4 Fib and V.WAP on dashboard tree narrative?

[B] H1 ANALYSIS (Agent L2 — Anchor)
  - Current VWAP swing direction (BULL / BEAR label on chart)?
  - H1 structure: last BOS direction?
  - Strong High / Weak High / Strong Low / Weak Low marked?
  - Premium / Discount / Equilibrium: where is price?
  - PDH / PDL: Above / Inside / Below?

[C] M15 TACTICAL MAP (Agent L3 — Filter)
  - BSL nearest: ___  SSL nearest: ___
  - VAH: ___  VAL: ___  POC: ___
  - M15 structure: last BOS direction?
  - Any CHoCH in last 8 hours?
  - Liquidity: has BSL or SSL been swept in last session?

[D] AGENT DASHBOARD READ (60-second protocol)
  - Score: ___/15
  - Sync phase: _______________
  - Effective bias: _______________
  - ATR: High / Med / Low
  - M15 tree row: Regime / V.WAP / Fib direction

[E] SESSION BIAS — WRITTEN DECISION
  - My bias for this session: BULL / BEAR / NEUTRAL
  - Primary target (where is price going?): ___________
  - Invalidation level (if price does this, bias is wrong): ___________
  - Signal type I am looking for: SYNC / LOCAL / None (waiting)
  - Max risk for this session: $___________

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
PHASE 2: ACTIVE SESSION (10:00 - 17:00 EAT)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[F] ON SIGNAL FIRE
  - Run 6-check confluence gate (2-3 minutes)
  - Fill execution card
  - EXECUTE / SKIP / WAIT decision

[G] TRADE ACTIVE
  - Let the trade work. Do not touch SL before TP1.
  - At TP1: move SL to breakeven
  - At TP2: trail or take partial (multi-lot operators)
  - At TP3: Holder Mode active — VWAP trail takes over
  - No new entries on the same asset while a trade is active (default rule)

[H] NEW SIGNAL WHILE TRADE IS ACTIVE
  - If running in same direction: monitor only (no add)
  - If running in opposite direction: confirm — is the agent reversing?
  - If score drops below ±4: consider manual close of active trade

[I] MID-SESSION CHECK (12:00 EAT)
  - Dashboard 60-second read
  - Is the active trade progressing?
  - Has M15 structure changed since entry?
  - Is ATR still sufficient?

[J] NY OPEN (15:00 EAT)
  - New liquidity event. Price often sweeps Asian highs or lows.
  - Re-run pre-session lite analysis: has the bias held from 05:30?
  - Watch for score to increase as NY volume adds confluence

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
PHASE 3: POST-SESSION (17:00 EAT)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[K] SESSION CLOSE
  - Close any remaining trades before end of NY session (if not in Holder Mode)
  - Dashboard today section: note Sigs / Blocked / TP hits / SL count
  - Net pips and P&L for the day

[L] BRIEF AUTOPSY
  - Any SL hit? Read the autopsy. Write one sentence.
  - Did the session bias hold? If not — what changed and when?
  - One improvement for tomorrow.
```

### The Live Stream Analysis Format

```

12:00 EAT — CHECK-IN CLIP (10 minutes)
   1. M15 update: has structure changed?

   2. State P&L (dollar amount, points)
```

## DAY 4 — THURSDAY | RISK: ACCOUNT PROGRESSION AND THE PF GATE

### The Performance Benchmark

The proven benchmark period for ADSA v7.0:
```
Profit Factor: 6.32 (ELITE ✅)
Net points: +1,448 pts
Record: 12W / 3L
Period: 12 weeks
```

This is not a promise. This is a benchmark that tells the operator what a healthy system looks like. The dashboard shows daily PF in real time.

### Daily PF Gate

The operator's daily performance rules:

```
PF ≥ 2.5 (ELITE) → day is operating at benchmark standard
PF ≥ 2.0 (STRONG) → day is acceptable, keep trading
PF ≥ 1.5 (GOOD) → continue, but review any losses
PF ≥ 1.0 (MARGINAL) → stop adding positions, review the day
PF < 1.0 (NEGATIVE) → STOP. Close any active trades.
                        Review before trading again tomorrow.
                        Do not attempt to "trade out of" a negative day.
```

### Account Sizing Progression

Starting operators should follow this progression:

```
LEVEL 1 — Observation Mode (Weeks 1-2 of masterclass)
  No live trades. Watch only. Fill cards manually.
  Learn: dashboard reads, card fills, gate decisions.
  Capital committed: $0

LEVEL 2 — Micro Mode (Weeks 3-4 of masterclass)
  Minimum lotsize trades only. No position sizing.
  XAUUSD: 0.01 lot. GBPUSD: 0.01 lot.
  Daily risk cap: $15
  Capital required: ≥ $100

LEVEL 3 — Risk Model Mode (Week 5 and beyond)
  Full Platinum Risk Model. Position size from formula.
  Daily risk cap: operator-defined (e.g., 3% of account)
  Risk per trade: $15-$50 depending on account size
  Capital required: ≥ $300

LEVEL 4 — Full Operator Mode (post-masterclass)
  Multiple assets on watchlist.
  Telegram broadcast running.
  Prop firm scaling or live account compound protocol.
```

### Mastery Is Not Perfection

No operator is perfect. The agent is not perfect. Markets are not predictable.

Mastery is:
- Consistent process execution (the card, the gate, the log)
- Correct capital management (always dollars first, never lots first)
- Honest autopsy every week (no denial, no blame, only learning)
- Discipline to skip low-probability setups
- Patience to wait for high-probability setups

The agent has done this for months in live markets. The operator's job is to match the agent's discipline with human contextual intelligence. Together: the formula.

**Analysis + Capital + Execution = Success.**
